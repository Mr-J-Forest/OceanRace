# 单一数据集涡旋标注与训练执行手册

## 1. 唯一源数据

- `data/processed/eddy_detection/19930101_20241231_clean.nc`

说明：
- 本任务只有一个源数据文件。
- train/val/test 只是对该文件按时间切分的索引，不是新增数据集。

## 2. 标准标签格式（训练可直接读取）

建议标签文件：
- `data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask_bg0.nc`

最小字段：
- `eddy_mask(time, latitude, longitude)`
- 编码：`0=background, 1=cyclonic, 2=anticyclonic`

## 3. 先跑 META4 标签流程（当前仓库现成）

当前仓库已具备 META4 风格标签脚本，推荐直接建立可训练闭环：

```bash
cd .
python scripts/02c_generate_meta4_labels.py \
  --clean-nc data/processed/eddy_detection/19930101_20241231_clean.nc \
  --out-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask.nc

python scripts/02h_fix_meta4_mask_background.py \
  --input-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask.nc \
  --output-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask_bg0.nc
```

如果你手里的是对象级结果或需要显式像素标签，再继续执行：

```bash
python scripts/02j_objects_to_mask_parallel.py \
  --clean-nc data/processed/eddy_detection/19930101_20241231_clean.nc \
  --objects-nc data/processed/eddy_detection/labels/19930101_20241231_objects_meta4.nc \
  --out-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask_bg0.nc
```

## 4. 对单文件按时间切分

```bash
python scripts/02b_split_eddy_merged_by_time.py \
  --clean-nc data/processed/eddy_detection/19930101_20241231_clean.nc \
  --label-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask_bg0.nc \
  --out data/processed/splits/eddy_merged_time.json
```

## 5. 神经网络训练（边界定位 + 形态分割）

```bash
python scripts/03_train_eddy.py \
  --time-split-manifest data/processed/splits/eddy_merged_time.json \
  --clean-nc data/processed/eddy_detection/19930101_20241231_clean.nc \
  --label-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4_mask_bg0.nc \
  --epochs 8 \
  --batch-size 4 \
  --out-dir outputs/eddy_detection/single_nc_train
```

## 6. 关于 META4.0 DT 标签规范

当前推荐状态：
- 第3步直接采用 META4.0 DT 方法对象级标签（闭合轮廓 + 形状/振幅/像素约束 + 网络化）。
- 通过 `02h_fix_meta4_mask_background.py` 保证背景为显式 0，避免 NaN 背景影响训练。
- 需要训练像素级分割时，再用 `02j_objects_to_mask_parallel.py` 转成 mask。
- 训练脚本与数据集读取接口可复用，不需要改动整体训练主干。

备注：
- 当前环境在直接加载 `py-eddy-tracker` 时存在依赖版本兼容问题（主要是 numpy/numba/zarr 组合约束），建议为 META 标注单独创建隔离环境执行，再把输出标签文件回写到本工程路径。
- 可使用本仓库提供的依赖清单：`requirements-meta4-labeler.txt`。

使用 META4 风格标注脚本（新增）：

```bash
python scripts/02c_generate_meta4_labels.py \
  --clean-nc data/processed/eddy_detection/19930101_20241231_clean.nc \
  --out-nc data/processed/eddy_detection/labels/19930101_20241231_label_meta4.nc
```

先做小窗口试跑（建议）：

## 7. 兼容性修复记录（2026-04）

本节记录一次已验证可复现的环境修复，用于后续排障复用。

现象（典型报错）：
- `ImportError: cannot import name 'float_' from 'numpy'`
- `AttributeError: 'QuadContourSet' object has no attribute 'collections'`

根因：
- `py-eddy-tracker` 当前源码对依赖版本有约束：
  - 需要 `numpy<2`（源码使用 `numpy.float_`，在 numpy>=2 被移除）。
  - 需要 `matplotlib<3.8`（轮廓对象接口兼容性）。

推荐修复（隔离环境，优先）：

```bash
python -m venv .venv-meta4
source .venv-meta4/bin/activate
python -m pip install -U pip
python -m pip install -r requirements-meta4-labeler.txt
python -m pip install xarray
```

应急修复（系统环境，谨慎使用）：

```bash
python -m pip install --user "numpy<2" "matplotlib<3.8" pint polygon3 scipy
```

说明：
- 系统环境应急修复用于临时跑通流程，可能与其它包（如某些版本的 opencv-python-headless）存在版本冲突提示。
- 生产与长期开发建议始终使用隔离环境（`.venv-meta4`）执行 META4 标注链路。
