#### 1.3.5 【风浪异常评估】风浪多变量输入与分支特征空间解耦

引入动机（机制：风浪多变量输入与分支特征空间解耦机制）：

本节引入“风场与浪场双分支解耦建模”机制，核心做法是将六变量按物理语义拆分为风分支（u10/v10/wind_speed）与浪分支（swh/mwp/mwd），分别进入独立编码器学习潜在表示。在黄渤海风浪耦合场景下，该机制主要解决直接拼接输入导致的跨变量语义污染问题，避免把风速突增误判为波场异常或把波高抬升错误归因于风向扰动，从而提升异常识别的稳定性与可解释性。

算法原理：

系统将输入按物理语义拆分为两组：风场分支输入为 Xo=[u10,v10,wind_speed]，浪场分支输入为 Xw=[swh,mwp,mwd]。分别建立独立编码器 Eo 与 Ew，获得解耦潜变量 zo 与 zw：

zo=Eo(Xo), zw=Ew(Xw)

在样本构建阶段同步生成有效掩膜 Mo 与 Mw，对缺测和陆地区域做加权屏蔽，降低无效像元对梯度更新的污染。该策略保证了风浪异常信号在特征空间中可分离、可解释。

代码级实现要点：

1. 数据加载器逐时刻读取六变量并分别堆叠为 oper_x 与 wave_x。
2. 对每个变量生成 valid mask，后续损失按 mask 加权。
3. 每个样本保留 timestamp 与 time_index，支撑后续事件时序关联。

代码实现（节选）：

```python
def __getitem__(self, idx: int) -> dict[str, Any]:

	ds_o, ds_w = self._get_pair_ds(pair_idx)

	o_u10, m_u10 = self._get_var(ds_o, "u10", t, default_shape=o_shape)
	o_v10, m_v10 = self._get_var(ds_o, "v10", t, default_shape=o_shape)
	o_ws, m_ws = self._get_var(ds_o, "wind_speed", t, default_shape=o_shape)

	w_swh, m_swh = self._get_var(ds_w, "swh", t, default_shape=w_shape)
	w_mwp, m_mwp = self._get_var(ds_w, "mwp", t, default_shape=w_shape)
	w_mwd, m_mwd = self._get_var(ds_w, "mwd", t, default_shape=w_shape)

	oper_x = torch.stack([o_u10, o_v10, o_ws], dim=0)
	wave_x = torch.stack([w_swh, w_mwp, w_mwd], dim=0)
```

参数优化与创新点：

1. 在保证信息完备的前提下采用三通道对三通道对称结构，减少参数规模并稳定训练。
2. 通过缺测掩膜参与损失计算，提升沿岸和边界网格的鲁棒性。
3. 解耦输入与分支结构天然支持后续跨模态一致性约束，形成可扩展异常评估框架。

#### 1.3.6 【风浪异常评估】交叉重构与物理一致性约束

引入动机（机制：Cross-Modal 交叉重构与物理一致性约束机制）：

本节引入“交叉重构 + 一致性约束”机制，核心做法是在主重构之外增加跨分支重构路径，并通过潜空间一致性损失联合优化。在当前风浪异常评估场景下，该机制主要解决仅靠单分支重构时对非物理耦合异常不敏感的问题，使模型能够显式利用风浪互约束关系，放大耦合失衡区域的误差响应，提升异常定位准确性。

算法原理：

主重构路径：

Xo_main=Do(zo), Xw_main=Dw(zw)

交叉重构路径：

Xo_cross=Xo_main+Bo_from_w(zw)
Xw_cross=Xw_main+Bw_from_o(zo)

其中 Bo_from_w 与 Bw_from_o 为跨模态映射头。训练损失由主重构损失、交叉重构损失和潜空间一致性损失组成：

L=L_main+lambda_cross*L_cross+lambda_fuse*L_fuse

L_fuse 用于约束 zo 与 zw 的统计偏离，抑制跨分支漂移。

代码级实现要点：

1. 前向输出同时包含 oper_recon、wave_recon、oper_cross_recon、wave_cross_recon。
2. 训练阶段分别计算 loss_main、loss_cross、loss_fuse 并加权求和。
3. 损失函数采用 mask 加权 MSE，确保异常分数聚焦有效海域。

代码实现（节选）：

```python
def masked_mse(pred: torch.Tensor, target: torch.Tensor, valid: torch.Tensor, eps: float = 1e-6) -> torch.Tensor:
	diff = (pred - target) ** 2
	w = valid.float()
	return (diff * w).sum() / (w.sum() + eps)

out = model(oper_x, wave_x)
loss_oper_main = masked_mse(out["oper_recon"], oper_x, oper_v)
loss_wave_main = masked_mse(out["wave_recon"], wave_x, wave_v)
loss_main = 0.5 * loss_oper_main + 0.5 * loss_wave_main

oper_cross = out.get("oper_cross_recon", out["oper_recon"])
wave_cross = out.get("wave_cross_recon", out["wave_recon"])
loss_oper_cross = masked_mse(oper_cross, oper_x, oper_v)
loss_wave_cross = masked_mse(wave_cross, wave_x, wave_v)
loss_cross = 0.5 * loss_oper_cross + 0.5 * loss_wave_cross

loss_fuse = torch.mean((out["oper_z"] - out["wave_z"]) ** 2)
loss = loss_main + lambda_cross * loss_cross + lambda_fuse * loss_fuse
```

参数优化与创新点：

1. 通过 lambda_cross 控制交叉约束强度，避免过强耦合导致训练不稳。
2. 通过 lambda_fuse 控制潜空间一致性，避免双分支表达完全独立。
3. 该设计在不增加复杂推理图的情况下，显著提升对非物理耦合异常的敏感度。

#### 1.3.7 【风浪异常评估】多源双分支交叉模态融合自编码器（Dual-Branch Cross-Modal AE）设计

引入动机（机制：Dual-Branch Cross-Modal AE + Fusion Gate 动态融合机制）：

本节引入“多源双分支交叉模态融合自编码器”机制，核心做法是通过双分支卷积 AE 提取风浪潜变量，并使用 Fusion Gate 对两类潜变量进行动态加权融合。在当前海洋灾害识别场景下，该机制主要解决风浪异常主导关系随时空变化而切换的问题，避免固定融合策略在不同海况下失效，从而增强模型对复杂耦合异常的泛化能力。

网络结构设计：

1. 每个分支采用 编码器-瓶颈-解码器 结构，编码端两级下采样，解码端两级上采样。
2. 瓶颈后做全局池化，得到潜变量 zo、zw。
3. 融合门控模块对拼接潜变量进行非线性映射，输出融合权重 g：

g=sigmoid(MLP([zo;zw]))
zf=g*zo+(1-g)*zw

4. 通过跨模态映射头将另一分支潜变量注入当前分支重构，形成交叉重构输出。

代码级实现要点：

1. 双分支主干结构一致，便于参数管理与部署。
2. 前向接口统一返回潜变量、融合门控与重构结果，便于训练、评估、可视化共用。
3. 输出保持与输入同尺寸，保证与现有前端网格渲染链路兼容。

代码实现（节选）：

```python
class DualBranchAutoEncoder(nn.Module):
	def __init__(self, oper_channels: int = 3, wave_channels: int = 3, base_channels: int = 24):
		super().__init__()
		latent_dim = base_channels * 4
		self.oper_branch = _BranchAE(in_channels=oper_channels, base_channels=base_channels)
		self.wave_branch = _BranchAE(in_channels=wave_channels, base_channels=base_channels)
		self.fusion_gate = nn.Sequential(
			nn.Linear(latent_dim * 2, latent_dim),
			nn.ReLU(inplace=True),
			nn.Linear(latent_dim, latent_dim),
			nn.Sigmoid(),
		)
		self.oper_from_wave = nn.Linear(latent_dim, oper_channels)
		self.wave_from_oper = nn.Linear(latent_dim, wave_channels)

	def forward(self, oper_x: torch.Tensor, wave_x: torch.Tensor) -> dict[str, torch.Tensor]:
		oper_recon, oper_z = self.oper_branch(oper_x)
		wave_recon, wave_z = self.wave_branch(wave_x)
		gate = self.fusion_gate(torch.cat([oper_z, wave_z], dim=1))
		fused_z = gate * oper_z + (1.0 - gate) * wave_z

		oper_cross_recon = oper_recon + self.oper_from_wave(wave_z).unsqueeze(-1).unsqueeze(-1)
		wave_cross_recon = wave_recon + self.wave_from_oper(oper_z).unsqueeze(-1).unsqueeze(-1)
		return {
			"oper_recon": oper_recon,
			"wave_recon": wave_recon,
			"oper_cross_recon": oper_cross_recon,
			"wave_cross_recon": wave_cross_recon,
			"oper_z": oper_z,
			"wave_z": wave_z,
			"fused_z": fused_z,
			"fusion_gate": gate,
		}
```

创新点与工程价值：

1. 动态门控使模型可根据样本状态自适应分配风浪权重，而非固定融合。
2. 交叉模态重构增强了异常放大机制，对非物理耦合更敏感。
3. 结构轻量、接口统一，便于在 FastAPI 推理端实现低延迟部署。

#### 1.3.8 【风浪异常评估】基于时间窗重叠的高鲁棒灾害事件时序关联机制

引入动机（机制：基于时间窗重叠的灾害事件时序关联机制）：

本节引入“时间窗硬交叉关联”机制，核心做法是将异常标志序列与台风事件生命周期窗口做 Intersection-over-Time 统计，直接输出命中事件与命中强度。在当前实时预警联动场景下，该机制主要解决 DTW 或检索式关联延迟高、工程链路重的问题，以更低计算复杂度完成大范围时序关联，满足在线分析与快速发布需求。

算法原理：

系统采用 时间窗硬交叉 策略：

1. 先由重构误差与阈值生成异常标志序列 flags。
2. 提取异常时刻集合 Tanom。
3. 对每个事件窗口 [start,end] 计算命中点数：

Hit_i=sum(1(start_i<=t<=end_i), t in Tanom)

4. 当 Hit_i>0 时认定事件命中，输出 matched_events。

该方法的核心优点是线性复杂度、解释直观、可直接映射到预警业务规则。

与标签构建链路的协同：

事件窗口由 IBTrACS 轨迹自动生成：

1. 依据流域、风速阈值、经纬度范围筛选有效台风点。
2. 生成事件起止时间并合并重叠时间窗。
3. 与数据集 timestamp 对齐后生成 labels 与 events。

代码级实现要点：

1. 关联函数输入为 timestamps、flags、events 三元组，输出命中事件统计。
2. 输出字段包括 num_matched_events、matched_events、anomaly_points，可直接用于前端展示和预警简报。
3. 在后端采用轻量缓存后，重复查询可快速响应，满足交互式分析。

代码实现（节选）：

```python
def associate_events(timestamps: list[int | float | None], flags: np.ndarray, events: list[dict[str, Any]]) -> dict[str, Any]:
	anom_ts = [int(t) for t, f in zip(timestamps, flags.tolist()) if t is not None and int(f) == 1]
	hits: list[dict[str, Any]] = []
	for ev in events:
		start = int(ev["start"])
		end = int(ev["end"])
		cnt = sum(1 for t in anom_ts if start <= t <= end)
		if cnt > 0:
			hits.append({
				"name": ev.get("name", "event"),
				"start": start,
				"end": end,
				"anomaly_points": int(cnt),
			})
	return {
		"num_events": int(len(events)),
		"num_matched_events": int(len(hits)),
		"matched_events": hits,
	}
```

```python
def _build_events(ibtracs_csv: Path, ts_min: int, ts_max: int, *, basin: str, min_wind_kts: float,
				  lat_min: float, lat_max: float, lon_min: float, lon_max: float, padding_hours: int) -> list[dict[str, Any]]:
	tracks: dict[str, list[int]] = defaultdict(list)
	with ibtracs_csv.open("r", encoding="utf-8", newline="") as f:
		reader = csv.DictReader(f)
		for row in reader:
			ts = _to_ts(row.get("ISO_TIME"))
			if ts is None:
				continue
			if ts < ts_min - 86400 or ts > ts_max + 86400:
				continue
			if basin and str(row.get("BASIN", "")).strip() not in {"", basin}:
				continue

			lat = _to_float(row.get("LAT"))
			lon = _to_float(row.get("LON"))
			wind = _best_wind_kts(row)
			if lat is None or lon is None or wind is None or wind < float(min_wind_kts):
				continue

			lon = _norm_lon(lon)
			if not (lat_min <= lat <= lat_max and lon_min <= lon <= lon_max):
				continue

			sid = str(row.get("SID", "")).strip()
			if sid:
				tracks[sid].append(ts)
```
